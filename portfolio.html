GradeCalHub

GradeCalHub is a web application designed to help students efficiently calculate and track their academic grades. Built with a React front-end, it offers a responsive and intuitive user interface for seamless interaction. The backend, powered by Node.js, ensures robust data processing and secure communication between the client and server. Utilizing MongoDB Atlas for its database, GradeCalHub provides reliable data storage and retrieval, enabling users to easily manage their coursework and monitor their academic progress. This project demonstrates a full-stack development approach, highlighting skills in modern web technologies and cloud-based solutions.

CS 3642 Assignment #3

The 3rd assignment for my CS 3642 (Artificial Intelligence) class was to make a perceptron capable of training itself to solve the three black squares problem and then demonstrate it in another fashion. The problem I decided to demonstrate the capability of the perceptron was a movie prediction algorithm that would predict the odds a user of a streaming app would like a movie (in this example The Matrix) based off of movies they have previously watched. I ran 5 tests to determine the classification accuracy of the perceptron on the three black squares problem, each test has 16 possible states for a total of 80 states. Out of those 80 total states 78 were correct. With 2 of them being false positives that determine the 4 blocks were dark when only 2 of the squares were black and 2 were white. That being said, 78 / 80 is still a great success rate at a 97.75% classification Accuracy. It is hard to calculate the accuracy of the movie prediction perceptron since whether or not a person would enjoy a movie is subjective. So, we cannot look at the result and objectively say whether or not the weights are accurate. But we can look at them and say that the results show a clear trend, which is exactly the outcome I was are trying to produce from the perceptron.

Virtual Operating System

Semester project for CS 3502 where we created a virtual operating system made from scratch coded in the C++ programming language. The goal of this project was to simulate our own simple operating system so we could get a better idea of how real world operating systems operate and complete both simple and complex tasks. The operating system we created takes in basic instructions and completes operations based on said instructions through the form of processes. It is also capable of running multiple processes on multiple simulated processing cores, the number of which is dependent on the user. To properly accomplish this we had to create a virtual hard-drive, virtual primary memory, virtual processor, virtual bootloader, and a scheduler to place processes from our waiting queue onto an available processor. All of these components and a lot more come together to form a functioning virtual operating system.
